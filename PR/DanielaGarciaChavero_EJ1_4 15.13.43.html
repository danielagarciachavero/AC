K-means sobre el dataset ampliado de Shapes (Ejercicio 1.4)¶

En este notebook se utiliza el dataset de 100 elementos generado mediante oversampling sintético.
El objetivo es:

  1. Cargar el CSV desde GitHub.
  2. Preparar las variables categóricas para aplicar K-means.
  3. Calcular una propuesta de k óptima mediante el método del codo y la silueta.
  4. Aplicar el algoritmo K-means.
  5. Analizar los clusters resultantes.

4. Cálculo de la k óptima¶

Se calculan dos métricas para distintos valores de k:

  • Inercia: se usa para el método del codo. Valores más bajos indican clusters más compactos.
  • Coeficiente de silueta: mide la separación entre clusters. Valores más altos indican una mejor estructura de agrupación.

En este notebook se toma como k principal el valor que maximiza la silueta, y se muestra también la sugerencia del método del codo.

In [30]:
def detectar_codo(k_values, inercias):
    """Detecta el codo como el punto con mayor distancia a la línea entre extremos."""

    puntos = np.column_stack((k_values, inercias))

    primero = puntos[0]
    ultimo = puntos[-1]

    linea = ultimo - primero
    norma_linea = np.linalg.norm(linea)

    if norma_linea == 0:
        return int(k_values[0])

    distancias = []

    for punto in puntos:
        distancia = abs(
            linea[0] * (primero[1] - punto[1])
            - (primero[0] - punto[0]) * linea[1]
        ) / norma_linea

        distancias.append(distancia)

    return int(k_values[np.argmax(distancias)])


max_k = min(10, len(df) - 1)
k_values = list(range(1, max_k + 1))

inercias = []
siluetas = []

for k in k_values:
    modelo = KMeans(n_clusters=k, random_state=RANDOM_STATE, n_init=20)
    etiquetas = modelo.fit_predict(X_encoded)
    inercias.append(modelo.inertia_)

    if k >= 2:
        siluetas.append(silhouette_score(X_encoded, etiquetas))

k_values_silueta = k_values[1:]

k_codo = detectar_codo(np.array(k_values), np.array(inercias))
k_silueta = int(k_values_silueta[np.argmax(siluetas)])

print("k sugerida por el método del codo:", k_codo)
print("k sugerida por el coeficiente de silueta:", k_silueta)

9. Análisis de los resultados¶

La k óptima seleccionada fue 4, ya que tanto el método del codo como el coeficiente de silueta señalaron este valor como el más adecuado. Antes de aplicar el algoritmo K-means, las variables categóricas del dataset se transformaron mediante one-hot encoding, puesto que este algoritmo requiere trabajar con datos numéricos.

El algoritmo agrupó los datos principalmente en función de la forma geométrica. Los cuatro clústeres obtenidos se corresponden con los cuatro tipos de figuras presentes en el dataset: triángulos, círculos, rectángulos y cuadrados. Cada grupo contiene 25 elementos, lo que refleja la distribución equilibrada del dataset sintético. Por tanto, la separación realizada por K-means se explica sobre todo por la forma y, de manera secundaria, por atributos relacionados con ella, como el número de lados y la dirección.

En comparación con el criterio visual utilizado por el LLM, el resultado de K-means se aproxima más al agrupamiento basado en la geometría que al basado en el color. El LLM podía modificar su criterio según el número de conjuntos solicitado; por ejemplo, al pedir 2 clusters agrupó las figuras según colores cálidos y fríos. En cambio, K-means no separó los datos principalmente por color ni por tamaño, sino por la estructura geométrica de las figuras.

Los grupos obtenidos son coherentes con el dataset original y con el proceso de oversampling realizado. Esto se debe a que el dataset ampliado mantenía una distribución equilibrada de las cuatro formas. Además, se conservan las relaciones lógicas entre atributos: los triángulos tienen 3 lados, los círculos 0, los cuadrados y rectángulos 4, y las direcciones asignadas son compatibles con cada tipo de figura.
